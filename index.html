Prework Study Guide

HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content show to the user.
  • Hypertext Markup Language (HTML) is one of the three cornerstone languages used in web development: HTML, CSS, and JS
  • When create the structure for a webpage, you use a series of HTML elements to define the content and framework. These HTML elements are usually composed of an opening tag and a closing tag represented with angle brackets.
  • HTML elements are case-sensitive.
  • HTML attributes are always defined in the beginning element, never the closing element.
  • The head element element contains information, or metadata, that helps the browser to render the page correctly. Metadata is typically not seen by users, but it contains important information about accessibility, search engines, and performance. Elements included within the head are not visible when a user looks at a webpage. Inside the head element, we can use the following metadata tags.
    • '!DOCTYPE html'declaration is letting the browser know to expect HTML on the page.
    • The html opening tag, which represents the root, or start, of a HTML document.An HTML attribute defines additional information about an element. For example Inside a html tag is an attriube of a lang="en", which tells the browser the language for the webpage.
    • The meta element defines metadata such as the character set, description, keywords, author, and viewport.
    • The title element defines the title for a webpage. This element is required and is extremely important for Search Engine Optimization (SEO).
    • The style element defines the CSS styles associated with a website.
    • The base element defines the base URL for a webpage.
    • The link element connects an external resource to the HTML document.
    • The script element embeds client-side scripts in the HTML document.
  • Semantic HTML elements make your HTML easier to read by other developers and the browser. In semantic HTML elements, the name of the element clearly describes its meaning to both the browser and the developer by describing its content and the role it plays on a webpage. Some examples of semantic elements are the following.
    • form
    • table
    • article
  • Non-semantic elements tell you nothing about its content. Some examples of non- semantic elements are the following.
    • div
    • span
  • Semantic HTML also plays an important role in making sure your webpage is accessible to the widest variety of users, including those who use assistive technologies such as screen readers.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside of an element.
  • Cascading Style Sheets (CSS) is a programming language used by developers to define how webpage content is presented to users. CSS adds style to our webpage.
  • We use CSS to define style rules for specific elements of an HTML page and format them using a wide range of style options. CSS defines the colors, aesthetics, visual layout, and organization.

Git

  • The element (git status) checks what branch we are currently on.
  • The element (git checkout -b branch-name) creates a new branch and switches to it.
  • Git is a version control system that lets you manage (save) and keep track of each. version of your project.
  • Git commands always start with (git).
  • Git is great for tracking changes locally on your own computer, but to collaborate with others, you need to use a Git-based repository hosting service such as GitHub, Bitbucket and SourceForge. GitHub is free to share work and collaborate.

JavaScript

  • JavaScript (JS) is a high-level dynamic programming language that allows users to interact with websites. It is a tool that takes a static HTML/CSS webpage and makes it interactive.
  • Using JavaScript, you can power a website with effects, interactivity, data communication, and more.
  • The browser reads code in the order it appears in the file, so if the JavaScript loads first and it is supposed to affect the HTML that hasn't loaded yet, there could be problems. Placing JavaScript near the bottom of an HTML page is one way to accommodate this dependency.
  • Variables are a core feature of JavaScript. They are containers that store (data types) as values. You start by declaring a variable with the let keyword, followed by the name you give to the variable. (Example: let myVariable;)
    • You can name a variable nearly anything, but there are some restrictions.
    • You can assign a value to a variable by the (=) sign (but note that the quotation marks around the name mean something. Refer to JavaScript Variable list.). (Example: let myVariable = “Bob”;)
    • Variables may hold values that have different data types.
  • JavaScript Variables:
    • string- A sequence of text is known as a string. To signify that the value is a string, enclose it in single quote marks
    • number- Numbers don't have quotes around them.
    • Boolean- This is a True/False value. The words true and false are special keywords that don't need quote marks. (Example: let mVariable = true;)
    • Array- This is a structure that allows you to store multiple values in a single reference. (Example: let myVariable = [1,Bob,Steve,10] refers to each member of the array like this; myVariable [0], myVariable [1], etc. Make sure to put single quote marks around Bob and Steve in the array.)
    • Object- This can be anything. Everything in JavaScript is an object and can be stored in a variable. Keep this in mind as you learn.
  • A semicolon at the end of a line indicates where a statement ends. It is only required when you need to separate statements on a single line. However, some people believe it's good practice to have semicolons at the end of each statement.
  • JavaScript is case sensitive.
  • You can write comments in JavaScript just as you can in CSS: Comments are snippets of text that can be added along with code. The browser ignores text marked as comments.
  • If your comment contains no line breaks, it's an option to put it behind two slashes. Example //This is a comment
  • An operator is a mathematical symbol that produces a result based on two values (or variables).
    • Addition (+): Add two numbers together or combine two strings.
    • Subtraction(-): Muplitplication(*), Division(/)- These do what you'd expect them to do in basic math. Example: 8*2;//muplitple in JS in an asterisk.
    • Assignment (=): This assigns a value to a variable.
    • Strict equality (==): This performs a test to see if two values are equal. It returns a true/false (Boolean) result. (Example: let myVaruable = 3; myVariable ==4)
    • Not, Does-not-equal (!), (!==): This returns the logically opposite value of what it precedes. It turns a true into a false, etc. When it is used alongside the Equality operator, the negation operator tests whether two values are notequal.
  • Conditionals are code structures used to test if an expression returns true or not. A very common form of conditionals is the if...else statement. The expression inside the if () is the test. If this comparison returns true, the first block of code runs.If the comparison is not true, the second block of code—after the elsestatement—runs instead.
  • Functions are a way of packaging functionality that you wish to reuse. It's possible to define a body of code as a function that executes when you call the function name in your code. This is a good alternative to repeatedly writing the same code.
  • Functions often take arguments: bits of data they need to do their job. Arguments go inside the parentheses, separated by commas if there is more than one argument.
    • An anonymous function: doesn't have a name. There's an alternative way of writing anonymous functions, which we call an arrow function. An arrow function uses () => instead of function ()
    • The prompt() function: asks the user to enter data, and storing it in a variable after the user clicks OK.
    • The setItem() function: creates and store a data item. Example: localStorage.setItem(“name”, myName); used localStorage setItem() function to create and store a data item called ‘name’ and setting its value to the myName variable that contains the users name entry.
  • Null is a special value in JavaScript that refers to the absence of a value.
  • JavaScript is a dynamic language with dynamic types. Variables in JavaScript are not directly associated with any particular value type, and any variable can be assigned (and re-assigned) values of all types.JavaScript is also a weakly typed language, which means it allows implicit type conversion when an operation involves mismatched types, instead of throwing type errors.
  • The types in the JavaScript language consist of primitive values and objects. All types except objects define immutable values.
    • Primitive values: immutable datum (values that cant be changed once created) represented directly at the lowest level of the language. These are:
      • Boolean type: Represents a logical entity and can have two values: true and false.
      • Null type: Has exactly one value, null.
      • Undefined type: A variable that has not been assigned a value.
      • BigInt type (a Numeric type): You can safely store and operate on large integers even beyond the safe integer limit for Numbers.You can use operators with BigInts just like with Numbers although BitInts are not strictly equal to a number, only loosely so. BigInts cannot be operated on interchangeably with Numbers. Instead a TypeError will be thrown.
      • String type: It is a set of "elements" of 16-bit unsigned integer values used to represent textual data. Each element in the String occupies a position in the String therefore the length of a String is the number of elements in it. Although strings are immutable, it is possible to create another string based on an operation on the original string by using a substring of the original or a concatenation of two strings depending on what's needed.
      • Symbol type: A unique value that may be used as the key of an Object property.
      • Numeric type: ECMAScript has two built-in numeric types: Number and BigInt — along with the related value NaN (Not a Number). NaN is encountered when the result of an arithmetic operation cannot be expressed as a number.
    • Objects: A collection of properties. Objects are a mapping between keys and values (keys are strings or symbols and values can be anything). In some programming languages, Symbols are called "atoms". Functions are regular objects with the additional capability of being callable.
  • The Number type is a double-precision 64-bit binary format IEEE 754 value. Values outside the range ±(2^-1074 to 2^1024) are automatically converted:
    • Positive values greater than Number.MAX_VALUE (2^1024) are converted to +Infinity.
    • Positive values smaller than Number.MIN_VALUE (2^-1074 ) are converted to +0.
    • Negative values smaller than -Number.MAX_VALUE (-2^1024)are converted to -Infinity.
    • Negative values greater than -Number.MIN_VALUE (-2^-1074 )are converted to -0.
  • Data properties associate a key with a value. Its described by the following attributes:
    • value: The value retrieved by a get access of the property. Can be any JavaScript value.
    • writable: A boolean value indicating if the property can be changed with an assignment.
    • enumerable: A boolean value indicating if the property can be enumerated by a for...in loop.
    • configurable: A boolean value indicating if the property can be deleted, if it can be changed to an accessor property, and if it can have its attributes changed.
  • Accessor property: Associates a key with one of two accessor functions (get and set) to retrieve or store a value. An accessory property has the following attributes:
    • get: A function called with an empty argument list to retrieve the property value whenever a get access to the value is performed. See also getters. May be undefined.
    • set: A function called with an argument that contains the assigned value. Executed whenever a specified property is attempted to be changed.
    • enumerable
    • configurable
  • Arrays are regular objects for which there is a particular relationship between integer-keyed properties and the length property.
  • Typed Arrays is an umbrella term for a range of data structures. They present an array-like view of an underlying binary data buffer, and offer many methods that have similar semantics to the array counterparts.
  • Set and WeakSet are data structures that take object references as keys. They represent a set of objects.
  • Map and WeakMap are data structures that take object references as keys. They associate a value to an object.
  • JSON (JavaScript Object Notation) is a lightweight data-interchange format, derived from JavaScript, but used by many programming languages.
  • JSON builds universal data structures.
  • The (typeof) operator can help you to find the type of your variable.
